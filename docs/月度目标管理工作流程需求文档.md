# 月度目标管理工作流程需求文档

## 一、概述

### 1.1 项目背景
为了提升员工目标管理的清晰度和执行效率，建立"月-周-日"三级目标管理体系，让员工在月初制定月度目标和周目标，在日报填报时自动关联并追踪进度。

### 1.2 设计原则
- **复用现有日报表**: 不修改现有的 `daily_reports`、`daily_work_items` 等表
- **新增独立模块**: 创建新的月度目标相关表和功能
- **渐进式关联**: 月周目标先独立填报，日报时再关联项目任务

## 二、数据库设计

### 2.1 新增表清单

| 表名 | 说明 | 依赖 |
|------|------|------|
| monthly_goals | 月度目标表 | 无 |
| weekly_goals | 周目标表 | monthly_goals |
| daily_report_goals | 日报目标关联表 | daily_reports, weekly_goals |

### 2.2 表结构详细设计

#### 2.2.1 月度目标表 (monthly_goals)

```sql
CREATE TABLE IF NOT EXISTS public.monthly_goals
(
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,           -- 员工ID
    user_name VARCHAR(100),                  -- 员工姓名
    month VARCHAR(7) NOT NULL,              -- 月份 (格式: 2026-02)
    title VARCHAR(500) NOT NULL,            -- 目标标题
    content TEXT,                             -- 工作内容 (可多行)
    status VARCHAR(20) DEFAULT 'draft',      -- 状态: draft/published/completed
    progress_rate NUMERIC(5,2) DEFAULT 0,     -- 完成进度 0-100
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT false
);

-- 索引
CREATE INDEX idx_monthly_goals_user_month ON monthly_goals(user_id, month);
```

#### 2.2.2 周目标表 (weekly_goals)

```sql
CREATE TABLE IF NOT EXISTS public.weekly_goals
(
    id SERIAL PRIMARY KEY,
    goal_id INTEGER NOT NULL,                -- 关联的月度目标ID
    week_number INTEGER NOT NULL,            -- 周次 (1-5)
    title VARCHAR(500) NOT NULL,            -- 周目标标题
    content TEXT,                             -- 工作内容
    progress_rate NUMERIC(5,2) DEFAULT 0,     -- 完成进度 0-100
    status VARCHAR(20) DEFAULT 'pending',    -- 状态: pending/in_progress/completed
    start_date DATE,                          -- 周开始日期
    end_date DATE,                            -- 周结束日期
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT false,
    CONSTRAINT fk_weekly_goals_monthly_goal FOREIGN KEY (goal_id)
        REFERENCES public.monthly_goals(id) ON DELETE CASCADE
);

-- 索引
CREATE INDEX idx_weekly_goals_goal_id ON weekly_goals(goal_id);
```

#### 2.2.3 日报目标关联表 (daily_report_goals)

```sql
CREATE TABLE IF NOT EXISTS public.daily_report_goals
(
    id SERIAL PRIMARY KEY,
    daily_report_id INTEGER NOT NULL,        -- 日报ID (关联现有daily_reports表)
    weekly_goal_id INTEGER NOT NULL,         -- 周目标ID
    project_id VARCHAR(50),                  -- 项目ID (日报填报时关联)
    project_name VARCHAR(200),               -- 项目名称
    task_node_id VARCHAR(50),                -- 任务节点ID
    task_node_name VARCHAR(200),             -- 任务节点名称
    notes TEXT,                              -- 填报说明
    completion_rate NUMERIC(5,2) DEFAULT 0,  -- 该任务完成度
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT false,
    CONSTRAINT fk_daily_report_goals_daily_report FOREIGN KEY (daily_report_id)
        REFERENCES public.daily_reports(id) ON DELETE CASCADE,
    CONSTRAINT fk_daily_report_goals_weekly_goal FOREIGN KEY (weekly_goal_id)
        REFERENCES public.weekly_goals(id) ON DELETE CASCADE
);

-- 索引
CREATE INDEX idx_daily_report_goals_report_id ON daily_report_goals(daily_report_id);
CREATE INDEX idx_daily_report_goals_weekly_goal_id ON daily_report_goals(weekly_goal_id);
```

## 四、API接口设计

### 4.1 月度目标API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/monthly-goals | 获取当前用户的月度目标列表 |
| POST | /api/v1/monthly-goals | 创建月度目标 |
| GET | /api/v1/monthly-goals/{id} | 获取月度目标详情 |
| PUT | /api/v1/monthly-goals/{id} | 更新月度目标 |
| DELETE | /api/v1/monthly-goals/{id} | 删除月度目标 |
| PUT | /api/v1/monthly-goals/{id}/publish | 发布月度目标 |
| PUT | /api/v1/monthly-goals/{id}/progress | 更新完成进度 |

### 4.2 周目标API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/monthly-goals/{goal_id}/weeks | 获取某月度的所有周目标 |
| POST | /api/v1/monthly-goals/{goal_id}/weeks | 创建周目标 |
| PUT | /api/v1/weekly-goals/{id} | 更新周目标 |
| DELETE | /api/v1/weekly-goals/{id} | 删除周目标 |
| POST | /api/v1/weekly-goals/{id}/ai-recommend | AI推荐周目标拆解 (后期) |

### 4.3 日报目标关联API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/daily-reports/{id}/goals | 获取某日报的目标关联 |
| POST | /api/v1/daily-reports/{id}/goals | 关联周目标到日报 |
| PUT | /api/v1/daily-report-goals/{id} | 更新目标关联 |
| DELETE | /api/v1/daily-report-goals/{id} | 删除目标关联 |
| GET | /api/v1/weekly-goals/current | 获取本周目标 (用于日报填报) |

## 六、与现有系统集成（方案1：关联表方式）

### 6.1 设计选择
**采用方案1：通过关联表复用现有日报表，不修改现有表结构**

```
现有表: daily_reports (不动)
    │
    └── 通过 daily_report_goals 关联 ──→ weekly_goals
                                                  │
                                                  └── monthly_goals
```

### 6.2 复用现有日报表
- 使用现有的 `daily_reports` 表存储日报数据
- 使用现有的 `daily_work_items` 表存储工作事项
- 不修改现有表结构

### 6.3 目标关联
- 在 `daily_report_goals` 表中关联 `daily_reports.id`
- 填报时通过日期自动匹配周目标

### 6.4 进度计算逻辑

```
周目标进度 = 关联的日报工作事项完成情况 / 计划工作日 × 100%

月度目标进度 = (第1周进度 + 第2周进度 + 第3周进度 + 第4周进度) / 周数
```

## 八、注意事项

1. **数据隔离**: 月度目标和周目标只允许创建者查看和修改
2. **时间判断**: 根据当前日期自动计算属于第几周
3. **进度更新**: 提交日报时自动更新周目标进度
4. **历史数据**: 不需要迁移现有日报数据
5. **兼容性**: 现有的日报管理、评价、分析功能保持不变
