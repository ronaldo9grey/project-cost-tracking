# 腾讯云同步命令

## � 部署流程确认

**您的部署流程是正确的：**

```
本地代码提交 → GitHub推送 → 腾讯云备份 → 拉取新代码 → 重启服务
```

### 🔄 完整部署流程（共5步）

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1️⃣ | 本地commit | 将本地修改提交到Git |
| 2️⃣ | GitHub push | 推送到GitHub仓库 |
| 3️⃣ | 服务器备份 | 在腾讯云创建项目备份 |
| 4️⃣ | git pull | 腾讯云从GitHub拉取最新代码 |
| 5️⃣ | 重启服务 | 重启Docker容器 |

## 🚀 PowerShell 同步到腾讯云

### 方式一：使用PowerShell脚本（推荐）

```powershell
# 在项目根目录执行
.\sync_to_tencent.ps1
```

脚本会自动执行以下操作：
- ✅ 检查并提交本地代码到GitHub
- ✅ 连接腾讯云服务器
- ✅ 创建项目备份
- ✅ 拉取最新代码
- ✅ 重启Docker服务
- ✅ 验证部署结果

### 方式二：手动命令

```powershell
# ========== 第一步：本地提交到GitHub ==========
git add .
git commit -m "feat: 新功能描述"
git push origin main

# ========== 第二步：腾讯云备份 ==========
ssh ubuntu@175.178.40.53 << 'EOF'
# 创建备份目录
mkdir -p /var/www/web_project_backups

# 备份项目（带时间戳）
backup_time=$(date +%Y%m%d_%H%M%S)
cp -r /var/www/web_project /var/www/web_project_backups/backup_$backup_time

# 记录备份信息
echo "Backup created: $(date)" > /var/www/web_project_backups/backup_$backup_time/backup_info.txt
echo "Git commit: $(git -C /var/www/web_project log --oneline -1)" >> /var/www/web_project_backups/backup_$backup_time/backup_info.txt

echo "Backup created: /var/www/web_project_backups/backup_$backup_time"
EOF

# ========== 第三步：拉取最新代码 ==========
ssh ubuntu@175.178.40.53 "cd /var/www/web_project && git pull origin main"

# ========== 第四步：重启服务 ==========
ssh ubuntu@175.178.40.53 "cd /var/www/web_project && docker-compose restart"

# ========== 第五步：验证 ==========
ssh ubuntu@175.178.40.53 "cd /var/www/web_project && docker-compose ps"
```

---

## 📁 备份说明

### 备份位置
- **备份目录**: `/var/www/web_project_backups/`
- **备份命名**: `backup_YYYYMMDD_HHMMSS`

### 恢复备份
```bash
# SSH连接到服务器
ssh ubuntu@175.178.40.53

# 进入项目目录
cd /var/www/web_project

# 删除当前文件
rm -rf *

# 从备份恢复
backup_folder="/var/www/web_project_backups/backup_20260212_123456"  # 替换为实际备份文件夹
cp -r $backup_folder/* .

# 重启服务
docker-compose restart
```

## ⚠️ 注意事项

- ✅ 备份在部署前自动执行
- ✅ 每次部署都会创建带时间戳的新备份
- ✅ 建议定期清理旧的备份文件节省空间
- ⚠️ 部署前确保SSH密钥已配置
